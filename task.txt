Проект будет оцениваться по 10-бальной шкале!
1. Цель работы
Закрепить навыки объектно-ориентированного программирования, работы со стандартной библиотекой STL, файлового ввода-вывода, проектирования модульных приложений и обработки пользовательского ввода в среде C++.
2. Постановка задачи
Разработать консольное приложение для автоматизации учёта товаров на складе. Программа должна обеспечивать добавление, редактирование, удаление записей, поиск и фильтрацию, сортировку, расчёт аналитических показателей, а также сохранение и загрузку данных в текстовом формате TXT.
3. Технические требования
•	Язык программирования: C++17 или выше
•	Компилятор: Любой
•	Хранение данных в памяти: std::vector или std::list
•	Формат файла данных: TXT с разделителем
•	Обработка ошибок ввода: без аварийного завершения программы; можно сделать обработку ошибок с завершением программы со снижением на 2 балла
•	Стиль кода: соответствие любому официальному стилю кода на выбор
4. Функциональные требования, усложнение - подключение SQL (по желанию, +3 балла к работе)
1.	Управление данными (CRUD):
o	Добавление товара с уникальным идентификатором
o	Редактирование существующей записи
o	Удаление товара по ID или названию
o	Валидация: цена ≥ 0, количество ≥ 0, непустые строки, уникальный ID
2.	Поиск и фильтрация:
o	По названию (частичное совпадение, регистронезависимо)
o	По категории
o	По диапазону цен (от–до)
3.	Сортировка:
o	По цене (возрастание/убывание)
o	По количеству на складе
o	По алфавиту (название)
4.	Аналитика:
o	Общая стоимость всех товаров на складе
o	Вывод списка товаров с остатком ≤ 5 шт.
o	Топ-3 самых дорогих позиции
5.	Работа с файлами:
o	Автоматическая загрузка data/products.txt при старте
o	Ручное сохранение изменений по команде пользователя
o	Формат строки: id,name,category,price,quantity
6.	Интерфейс:
o	Консольное текстовое меню с циклической навигацией
o	Корректное отображение кириллицы в Windows (SetConsoleOutputCP(65001) или аналог)

    6. Сдача и оформление
Работа на проверку принимается только через платформы удаленных репозиториев (GitHub, GitLab и т.д.). Репозиторий должен быть открытым и содержать не менее двух коммитов и обязательно содержать README.md.
